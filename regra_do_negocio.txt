Cenário

Uma loja de e-commerce deseja criar e gerenciar um banco de dados para apoiar suas operações e fornecer dados consistentes para análises. Esse banco de dados será projetado para armazenar informações de clientes, produtos, pedidos, pagamentos e campanhas de marketing. Além disso, será populado com dados simulados para validação e testes.

Objetivos do Projeto:

    1 - Criar um banco de dados relacional para armazenar as operações de e-commerce.
    2 - Populá-lo com dados fictícios para simular cenários reais.
    3 - Fornecer uma base consistente para futuras análises de dados e desenvolvimento de aplicações.


Regras de Negócio:

 1 - Histórico de Dados: Manter o histórico de pedidos e pagamentos nos últimos 3 anos.

Categorias de Clientes:
    1 - Bronze: Clientes que gastaram até $500.
    2 - Prata: Clientes que gastaram entre $501 e $2000.
    3 - Ouro: Clientes que gastaram acima de $2000.
